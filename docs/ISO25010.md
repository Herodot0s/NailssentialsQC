# ISO/IEC 25010 Software Product Quality Assessment

## NailssentialsQC — Operational Feasibility

---

### Introduction

This section evaluates the operational feasibility of the NailssentialsQC system using the ISO/IEC 25010:2011 Software Product Quality model. The ISO 25010 standard defines eight quality characteristics that collectively describe the degree to which a software product satisfies the needs of its stakeholders when used under specified conditions. By mapping the system's implemented features and architectural decisions against these quality characteristics, this assessment demonstrates that the NailssentialsQC system is operationally viable for deployment in a real salon environment.

The eight quality characteristics defined by ISO 25010 are: Functional Suitability, Performance Efficiency, Compatibility, Usability, Reliability, Security, Maintainability, and Portability. Each characteristic is discussed below in relation to the NailssentialsQC system's design and implementation.

---

### 1. Functional Suitability

Functional suitability measures the degree to which the software provides functions that meet stated and implied needs when used under specified conditions. This characteristic is subdivided into functional completeness, functional correctness, and functional appropriateness.

**Functional Completeness.** The NailssentialsQC system addresses the complete operational workflow of a nail salon. The system covers appointment booking with service selection and time-slot allocation, staff attendance tracking with clock-in/clock-out functionality, automated commission computation based on completed appointments, weekly payroll generation with tiered commission rates and categorized deductions, a content management system for the salon's public-facing landing page, service package bundling for promotional offerings, and analytics dashboards for revenue, staff performance, and customer retention metrics. These features collectively address all identified functional requirements across the three user roles (customer, staff, manager).

**Functional Correctness.** The system ensures computational accuracy through the use of Decimal data types with 10-digit precision and 2 decimal places for all monetary calculations, preventing floating-point rounding errors in commission and payroll computations. Payroll calculation results were validated against manually computed spreadsheet values during user acceptance testing to confirm correctness. The weekly calculation engine aggregates daily performance data per staff member, applies tiered commission rates (8% and 20% based on performance thresholds), and subtracts categorized deductions to produce accurate net pay figures.

**Functional Appropriateness.** The system's features are specifically tailored to the salon's operational context. For instance, the appointment system supports walk-in logging alongside scheduled bookings, reflecting real salon operations. The payroll deduction categories (Cash Advance, Loan, Uniform, Reloan, Lates/Early Out) mirror the salon's existing manual tracking categories, ensuring that the system aligns with established business practices rather than imposing unfamiliar workflows.

---

### 2. Performance Efficiency

Performance efficiency represents the performance relative to the amount of resources used under stated conditions. Its sub-characteristics include time behavior, resource utilization, and capacity.

**Time Behavior.** The system was optimized for responsive performance through several measures. Database queries utilize composite indexes on frequently accessed field combinations (e.g., staff_id with date for attendance lookups, appointment_date with status for scheduling views), reducing query latency. Payroll generation employs parallel database operations using Promise.all to aggregate data from multiple tables concurrently rather than sequentially. The frontend leverages React Query with a 10-minute stale time for CMS content, minimizing redundant API calls for data that changes infrequently while ensuring that updates propagate within an acceptable window.

**Resource Utilization.** The backend processes API requests through Express.js, which operates on a single-threaded event loop suitable for the I/O-bound nature of salon operations. Database connections are managed through the Prisma client's built-in connection pooling, preventing resource exhaustion under concurrent access. The frontend build is optimized through Vite's tree-shaking and code-splitting capabilities, delivering only the JavaScript required for each page view.

**Capacity.** The system is designed to handle the operational scale of a single salon location, with the PostgreSQL database capable of scaling to accommodate growth in appointment records, staff payroll history, and customer profiles without schema modification.

---

### 3. Compatibility

Compatibility describes the degree to which a product can exchange information with other products and perform its required functions while sharing the same hardware or software environment. It encompasses co-existence and interoperability.

**Co-existence.** The system operates as a web application accessible through standard browsers without requiring additional software installation on client devices. The backend runs independently as an Express.js server that does not interfere with other services on the host environment. Docker Compose isolates the development database to prevent conflicts with other local applications.

**Interoperability.** The system integrates with several third-party services through standard APIs. Clerk provides authentication management with role-based metadata synchronization. Vercel Blob and Cloudinary handle file storage for gallery images and profile pictures, respectively. The payroll system produces Excel-formatted exports compatible with the salon's existing spreadsheet workflows, ensuring seamless data exchange with external tools already in use by the management.

---

### 4. Usability

Usability measures the degree to which a product can be used by specified users to achieve specified goals with effectiveness, efficiency, and satisfaction. Its sub-characteristics include appropriateness recognizability, learnability, operability, user error protection, user interface aesthetics, and accessibility.

**Appropriateness Recognizability.** The system presents role-specific dashboards immediately upon login, allowing each user type to recognize the relevant features. Customers see their booking interface and appointment history, staff members see their attendance tracker and commission summary, and managers see payroll management, analytics, and content administration tools organized in a categorized sidebar.

**Learnability.** The interface follows established web application conventions with familiar patterns such as card-based layouts, sidebar navigation, dialog-based forms, and tabbed content areas. The manager dashboard employs a categorized sidebar grouping related functions (Operations, Analytics, Content, Payroll) to reduce cognitive load and help managers locate features intuitively.

**Operability.** The system supports efficient task completion through features such as quick login shortcuts for different user roles, walk-in appointment logging for immediate service recording, inline deduction editing within the payroll interface, and single-click Excel export for payroll reporting. The staff dashboard provides a swipeable check-in mechanism optimized for mobile interaction during salon operations.

**User Interface Aesthetics.** A premium design system was implemented using a curated color palette (Rausch #FF385C as the primary brand color), organic border-radius values (32px for containers, 12px for utility elements, 8px for data components), Playfair Display serif typography for hero sections paired with Inter sans-serif for body text, and framer-motion page transitions for smooth navigation between views. These design choices create a cohesive and professional visual identity appropriate for a service-oriented business.

**User Error Protection.** Input validation is implemented on both the frontend (form-level validation before submission) and backend (Zod schema validation on API endpoints) to prevent invalid data from entering the system. The payroll period locking mechanism prevents accidental modifications to finalized payroll records. Appointment scheduling enforces time-slot conflict detection to prevent double-booking of staff members.

---

### 5. Reliability

Reliability represents the degree to which a system performs specified functions under specified conditions for a specified period of time. It includes maturity, availability, fault tolerance, and recoverability.

**Maturity.** The system underwent three milestone releases (v1.0, v2.0, v2.1), with each release addressing defects discovered in prior iterations. The v1.0 milestone specifically targeted critical bug fixes, including JSX rendering errors, hardcoded credentials, type mismatches, and schedule upsert logic failures. Over 40 quick-task bug fixes were executed across the project lifecycle, progressively reducing the defect density of the codebase.

**Availability.** The production deployment on Vercel provides high-availability hosting with automatic scaling and global CDN distribution for the frontend. The Neon PostgreSQL cloud database includes managed availability guarantees. The system does not require manual server management or intervention to remain operational.

**Fault Tolerance.** The backend implements error handling middleware that catches unhandled exceptions and returns structured error responses rather than exposing stack traces. Database operations that involve multiple related writes (such as payroll generation involving payroll records, deduction entries, and commission aggregations) are wrapped in Prisma transactions to ensure atomicity — either all related changes succeed or none are applied, preventing data inconsistency.

**Recoverability.** Database schema evolution is managed through Prisma Migrate, which generates versioned migration files that allow both forward migration and rollback. Git version control provides a complete history of all code and schema changes, enabling recovery to any previous state. The payroll regeneration feature allows managers to recalculate payroll for a period while preserving manually entered deductions, supporting recovery from computation errors without data loss.

---

### 6. Security

Security describes the degree to which a product protects information and data so that persons or other products have the degree of data access appropriate to their types and levels of authorization. It encompasses confidentiality, integrity, non-repudiation, accountability, and authenticity.

**Confidentiality.** Authentication is implemented through JSON Web Tokens (JWT) with access and refresh token rotation. Passwords are hashed using bcrypt before storage, ensuring that plaintext credentials are never persisted. The JWT validation employs a fail-fast pattern that immediately rejects malformed or expired tokens before processing any request payload.

**Integrity.** Data integrity is enforced at the database level through foreign key constraints with cascading deletes where appropriate (e.g., deleting a User cascades to their profile and related tokens) and unique constraints on business-critical fields (e.g., username, email, phone, receipt numbers). All monetary fields use Decimal types rather than floating-point to prevent precision loss in financial calculations. Input validation through Zod schemas on payroll and appointment endpoints rejects malformed data before it reaches the database layer.

**Accountability.** The database schema includes a SystemLog model that records user actions with the associated user ID, action type, entity reference, and timestamp. This audit trail provides traceability for administrative actions within the system.

**Authenticity.** User identity is verified through Clerk-managed authentication in production, which supports multi-factor authentication capabilities. The system's auth middleware validates the JWT token on every protected request, ensuring that each API call is associated with a verified user identity. Refresh token rotation invalidates previously issued tokens upon each refresh cycle, preventing token reuse attacks.

**Non-repudiation.** All significant system actions (appointment creation, payroll generation, deduction entries) include timestamp fields and user association through foreign keys, creating an auditable record that links actions to authenticated users.

---

### 7. Maintainability

Maintainability represents the degree of effectiveness and efficiency with which a product can be modified. It includes modularity, reusability, analyzability, modifiability, and testability.

**Modularity.** The codebase is organized as a monorepo with clear separation between the frontend (React 19 + Vite) and backend (Express.js + TypeScript) applications. The backend follows a controller-route-middleware pattern, with each business domain (authentication, appointments, payroll, analytics, CMS) encapsulated in dedicated controller files. The frontend decomposes the UI into reusable components organized by feature domain (dashboard, home, packages, motion, analytics).

**Reusability.** Shared design primitives such as the PageTransition and AnimatedCard components are reused across all pages for consistent motion behavior. The categorized sidebar component serves both the manager and staff dashboards. API client functions are centralized in a shared API layer, enabling consistent endpoint access patterns across all frontend views.

**Analyzability.** TypeScript is used across both the frontend and backend, providing static type analysis that surfaces errors at compile time rather than runtime. The Prisma schema serves as a single source of truth for the database structure, making it straightforward to understand data relationships. Strict-mode TypeScript compilation is enforced, ensuring no implicit `any` types obscure the codebase.

**Modifiability.** The Prisma ORM decouples business logic from raw SQL, enabling schema modifications through migration files without rewriting query logic. The design token system (colors, radii, typography) is defined in global CSS variables, allowing visual identity changes from a single configuration point. The salary structure pattern (SalaryComponent, SalaryStructure, SalaryStructureAssignment) was designed to accommodate future compensation model changes without schema refactoring.

**Testability.** The test infrastructure includes Jest with Supertest for backend API testing and Vitest with React Testing Library for frontend component testing. The backend's controller-service separation enables unit testing of business logic independently from HTTP request handling. Test configuration files are maintained alongside the source code, and the Vitest environment was established in the first development sprint to support test-driven development throughout subsequent phases.

---

### 8. Portability

Portability describes the degree of effectiveness and efficiency with which a system can be transferred from one hardware, software, or operational environment to another. It encompasses adaptability, installability, and replaceability.

**Adaptability.** The frontend is built as a responsive web application using CSS media queries and Tailwind CSS responsive utilities, enabling the interface to adapt to different screen sizes from mobile devices to desktop monitors. The staff dashboard was specifically designed with a mobile-first approach, as salon staff primarily access the system through mobile devices during operations. The manager dashboard adapts between a full sidebar layout on desktop and a collapsible navigation on smaller screens.

**Installability.** The development environment can be provisioned through Docker Compose with a single command, which sets up the PostgreSQL database without manual configuration. The frontend and backend can be started concurrently using standard npm scripts. Production deployment to Vercel requires minimal configuration, with automatic builds triggered by Git pushes and environment variables managed through the platform's dashboard.

**Replaceability.** The system's modular architecture and standard technology choices facilitate component replacement if needed. The database layer is abstracted through Prisma ORM, allowing potential migration to alternative database engines (MySQL, SQLite) with schema adaptation. The authentication layer was migrated from a custom JWT implementation to Clerk during the project without disrupting other system components, demonstrating the architecture's tolerance for service replacement. File storage services (Vercel Blob, Cloudinary) communicate through standard REST APIs, enabling substitution with alternative storage providers.

---

### Summary

The evaluation against the eight ISO/IEC 25010 quality characteristics demonstrates that the NailssentialsQC system satisfies the criteria for operational feasibility. The system provides functionally complete and correct features aligned with salon business operations (Functional Suitability), performs efficiently through optimized queries and caching strategies (Performance Efficiency), integrates with third-party services and existing workflows (Compatibility), presents an intuitive and aesthetically refined interface for all user roles (Usability), maintains stability through progressive defect resolution and transactional data operations (Reliability), protects data through authentication, authorization, and input validation mechanisms (Security), supports ongoing modification through modular design and static type analysis (Maintainability), and can be deployed and adapted across different environments and device form factors (Portability). These qualities collectively support the conclusion that the system is operationally feasible for deployment in its intended salon management context.
