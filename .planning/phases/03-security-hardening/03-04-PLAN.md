<task type="auto">
  <name>Task 2: Add rate limiting middleware to authRoutes.ts</name>
  <files>backend/src/routes/authRoutes.ts</files>
  <read_first>backend/src/routes/authRoutes.ts</read_first>
  <action>
    In `backend/src/routes/authRoutes.ts`, add rate limiting to the auth endpoints.

    At the TOP of the file (after the imports, before the router definition), add the rate limiter instance:

      import rateLimit from 'express-rate-limit';

      const authRateLimiter = rateLimit({
        windowMs: 15 * 60 * 1000, // 15 minutes in milliseconds
        max: 5, // Limit each IP to 5 requests per windowMs
        message: {
          success: false,
          error: {
            code: 'RATE_LIMIT_EXCEEDED',
            message: 'Too many requests, please try again after 15 minutes',
          },
        },
        standardHeaders: true, // Return rate limit info in the `RateLimit-*` headers
        legacyHeaders: false, // Disable the `X-RateLimit-*` headers
      });

    Then apply the rate limiter to the login, register, and refresh routes by adding `authRateLimiter` as the FIRST middleware in the chain:

    Line 66 (change from):
      router.post('/register', registerValidation, register);

    To:
      router.post('/register', authRateLimiter, registerValidation, register);

    Line 67 (change from):
      router.post('/login', loginValidation, login);

    To:
      router.post('/login', authRateLimiter, loginValidation, login);

    Line 68 (change from):
      router.post('/refresh', refresh);

    To:
      router.post('/refresh', authRateLimiter, refresh);

    The logout and getMe routes do NOT need rate limiting (logout uses a token, getMe uses authenticateToken).
  </action>
  <verify>
    <automated>grep -c "authRateLimiter" backend/src/routes/authRoutes.ts</automated>
