## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, tab content padding |
| xl | 32px | Layout gaps between editor sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (landing page section bands) |

Exceptions:
- CMS form fields: 12px vertical gap (gap-3) between label+input pairs for compact density
- Contact Info section on landing page: 48px vertical padding (py-12) to match Signature Experience rhythm
- FAQ accordion items: 0px gap between items (border-b separators instead of spacing)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display | 36px | 400 (regular) | 1.2 |
| Heading | 24px | 400 (regular) | 1.3 |
| Body | 14px | 400 (regular) | 1.5 |
| Label | 10px | 700 (bold) | 1.2 |

**Font families:** Inter (sans-serif body/labels), Playfair Display (serif display/headings)

**Weights:** 400 (regular — body, display via `font-light` serif) and 700 (bold — labels, micro-copy)

**Phase 5 specific conventions:**
- CMS editor section headings: `font-serif text-xl font-light` (Playfair, 20px, weight 400) — slightly smaller than Display for admin context
- CMS form labels: `text-[10px] uppercase tracking-widest font-bold text-muted-foreground` — matches established NailssentialsQC label pattern
- CMS field descriptions/help text: `text-xs text-muted-foreground` (12px, Inter)
- `/policies` page title: `font-serif text-4xl font-light` — Display role
- FAQ question text: `text-base font-medium` (16px, weight 500, Inter)
- FAQ answer text: `text-sm text-muted-foreground leading-relaxed` (14px, Inter)
- Policy title: `font-serif text-2xl font-light` — Heading role
- Policy body: `text-sm text-muted-foreground leading-relaxed whitespace-pre-wrap` (14px, Inter, preserves line breaks)
- Contact Info labels: `text-xs tracking-[0.3em] text-muted-foreground uppercase` — matches Philosophy section pattern

## Layout Contract

### CMS Editor — "Content" View (D-11, D-12)

**Location:** New sidebar nav item in ManagerDashboard, registered as `activeView: 'content'`

**Structure:** Full-width content area (inherits Phase 4 sidebar+content layout)

| Zone | Behavior |
|------|----------|
| Tabs header | `Tabs` component with 3 triggers: "Landing Page" / "FAQ" / "Policies" |
| Tab content | Scrollable form area, `p-6` padding |

**Tab style:** Reuse Phase 4 tab pattern — `text-[10px] uppercase tracking-[0.2em] font-bold`, `border-b-2` active indicator in primary color

**Sidebar entry:**
- Icon: `FileText` from lucide-react
- Label: `Content`
- Position: After "Exhibit Gallery" in nav order
- Active state: `bg-primary text-white shadow-lg shadow-primary/20` (same as all other items)

### Landing Page Tab (D-01, D-02, D-10)

**Structure:** Vertical stack of collapsible Card sections

| Section Card | Fields | Layout |
|-------------|--------|--------|
| Hero | tagline (Input), headline (Input), subheadline (Textarea 3 rows), background image URL (Input), primary button label (Input) | Single column, `space-y-3` between fields |
| Signature Experience | label (Input), headline (Input), body paragraph (Textarea 4 rows), link label (Input) | Single column, `space-y-3` |
| Footer CTA | headline (Input), button label (Input) | Single column, `space-y-3` |
| Contact Information | phone (Input), address (Input), operating hours (Input), email (Input), Google Maps link (Input) | Single column, `space-y-3` |

Each Card has:
- `CardHeader` with serif heading (`font-serif text-lg font-light`) + muted description
- `CardContent` with form fields
- `CardFooter` with right-aligned "Save Changes" `Button` (default variant)
- Save button shows loading spinner (`Loader2 animate-spin`) during mutation

### FAQ Tab (D-03, D-05)

**Structure:** List of FAQ items + "Add FAQ" button

| Element | Spec |
|---------|------|
| FAQ list | Vertical stack of Cards, each with: title (Input), body (Textarea 4 rows), sort_order (Input type=number), is_active (Checkbox) |
| Each FAQ card | `CardHeader` with title preview + Edit/Delete icon buttons (right-aligned) |
| Add button | `Button variant="outline"` with `Plus` icon, full-width at bottom, `border-dashed` |
| Delete | Destructive `Button` with `Trash2` icon, triggers `window.confirm()` |
| Sort order | `Input type="number"` with `w-20` width, inline with is_active checkbox |

### Policies Tab (D-04)

**Structure:** Identical to FAQ tab layout (unified `SiteContent` model)

| Element | Spec |
|---------|------|
| Policy list | Vertical stack of Cards, each with: title (Input), body (Textarea 6 rows), sort_order (Input type=number), is_active (Checkbox) |
| Add button | Same as FAQ — `Button variant="outline"`, `Plus` icon, `border-dashed` |
| Delete | Same as FAQ — destructive with confirmation |

## Interaction Patterns

### CMS Editor Save Flow (D-10)
1. Manager navigates to "Content" in sidebar → ContentView renders with "Landing Page" tab active
2. Manager edits fields within a section Card (e.g., Hero)
3. Manager clicks "Save Changes" on that specific Card
4. Button shows `Loader2 animate-spin` + disabled state during API call
5. On success: green toast "Changes saved successfully." + React Query cache invalidation
6. On error: red toast "Failed to save changes. Please try again."
7. No autosave — explicit save per section (D-10)

### FAQ/Policy CRUD Flow (D-03, D-04)
1. Manager sees list of existing items as Cards
2. Click "+ Add FAQ Item" → new empty Card appears at bottom with inputs
3. Fill title + body + sort_order + is_active → click "Save" on that Card
4. To edit: modify fields inline → click "Save Changes"
5. To delete: click `Trash2` icon → `window.confirm("Delete this FAQ item? This cannot be undone.")` → API delete → card removed
6. Sort order determines display order on landing page and `/policies`

### Landing Page Content Hydration (D-08, D-09)
1. `Home.tsx` fetches `/api/cms/settings` via React Query with `staleTime: 10 * 60 * 1000`
2. Each component (Hero, Signature, Footer CTA, Contact Info) receives DB values as props
3. If a DB value is null/undefined, component falls back to current hardcoded string
4. Contact Info section: if ALL 5 fields are empty → section renders nothing
5. FAQ section: if 0 active FAQs → section renders nothing

### No Live Preview (D-13)
- CMS editor has no preview panel
- Manager saves changes, then opens landing page in new browser tab to review
- Matches existing ManageExhibits pattern
