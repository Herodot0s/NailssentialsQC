## Phase 1 Specific UI Notes

### BUG-01: Navbar DropdownMenu JSX Fix
**File:** `frontend/src/components/Navbar.tsx`
**Issue:** Missing space between component name and `render` prop causes JSX syntax errors
**Fix:** Add space before `render` prop — no visual changes
**Lines affected:**
- Line 105: `<DropdownMenuTrigger render={>` → `<DropdownMenuTrigger render={() => ...}>`
- Lines 130, 136, 143, 150: `<DropdownMenuItemrender=` → `<DropdownMenuItem render={...}>`
- Lines 206, 210-211: Same pattern in mobile menu

**Verification:** After fix, Navbar dropdown menu must render correctly without React JSX errors.

### BUG-03: ManagerDashboard Type Fix
**File:** `frontend/src/pages/ManagerDashboard.tsx`
**Issue:** `as any` cast on `setActiveView(item.id as any)`
**Fix:** Define `type ActiveView = 'analytics' | 'staff' | 'attendance' | 'deductions' | 'payroll' | 'reviews'` and use `as ActiveView` — no visual changes
