<action>
Create `backend/src/controllers/analyticsController.ts` with three exported async functions.

**1. `getStaffPerformance`:**
- Accept `startDate` and `endDate` query params (same validation pattern as `getHistoricalAnalytics`)
- Use `prisma.commission.groupBy({ by: ['staff_id'], where: { commission_date: { gte: start, lte: end } }, _sum: { base_amount: true, commission_amount: true }, _count: { id: true } })`
- Join with `prisma.staffProfile.findMany()` to get `full_name` per staff
- For category breakdown: query `prisma.commission.findMany({ where: { commission_date: { gte, lte } }, include: { service: { include: { category: true } }, staff: true } })`, then group in JS to produce `Record<staffId, Record<categoryName, number>>`
- Sort by revenue descending
- Return `{ success: true, data: StaffPerformanceStat[] }` where each item has: `{ staffId, fullName, revenue, commission, serviceCount, categoryBreakdown }`

**2. `getRetentionAnalytics`:**
- Accept `startDate` and `endDate` query params
- **Retention rate:** Find all unique `customer_id` values from completed appointments in range. For each, check if they have another completed appointment within 60 days after their last visit in the period. Use a batch approach: get all customer last-visit dates, then query appointments in the 60-day-forward window with `customer_id: { in: [...] }`.
- **New vs Returning:** Query `prisma.appointment.groupBy({ by: ['customer_id'], where: { status: 'completed' }, _min: { appointment_date: true } })` to get first-ever visit per customer. Customers whose first visit falls in the selected period = "New". Others = "Returning".
- **Trend:** Use `eachMonthOfInterval` from date-fns. For each month in the range, calculate the 60-day retention rate. Return `{ month: string, rate: number }[]`.
- **Top customers:** Group completed appointments by `customer_id` in range, count, sort desc, take 10. Join with `CustomerProfile` for `full_name`.
- Return `{ success: true, data: { retentionRate, totalCustomers, returningCustomers, newCustomers, trend, topCustomers } }`

**3. `getKpiSummary`:**
- Accept `startDate` and `endDate` query params for "this month" context
- `todayRevenue`: sum of completed transactions today (reuse `getDailySalesStats` logic with `prisma.transaction.aggregate`)
- `monthRevenue`: sum of completed transactions in the month (startDate to endDate)
- `activeStaff`: `prisma.staffProfile.count({ where: { is_available: true } })`
- `monthAppointments`: `prisma.appointment.count({ where: { status: 'completed', appointment_date: { gte: start, lte: end } } })`
- For trend indicators: query previous period (same duration offset backward) to calculate % change
- Return `{ success: true, data: { todayRevenue, monthRevenue, activeStaff, monthAppointments, todayRevenueTrend, monthRevenueTrend, monthAppointmentsTrend } }`

**Imports:**

