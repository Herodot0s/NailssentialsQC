- Unselected: `border border-primary/10 bg-[#fafaf9] text-foreground text-xs px-3 py-2 cursor-pointer hover:border-primary/30 transition-colors rounded-none`

- Click toggles service in/out of selection

**Summary section** (below all categories):
- `text-sm font-medium` — "Selected: {N} services • Total: ₱{sum}"
- If selection count < 2: `text-xs text-destructive` — "Select at least 2 services"

**Savings preview** (passed via optional `packagePrice` prop):
- If `packagePrice` is set and < sum: `text-xs text-success-color font-bold` — "Customer saves ₱{difference}"

**Dialog structure** — `Dialog` with `DialogContent className="sm:max-w-2xl"` (fullscreen on mobile):

**DialogHeader:**
- `DialogTitle`: `font-serif text-xl font-light` — "New Package" (create) / "Edit Package" (edit)

**Form sections** (scrollable body with `space-y-6`):

**Section 1 — Basic Info** (single column, `space-y-3`):
- Label: `text-[10px] uppercase tracking-widest font-bold text-muted-foreground` — "Package Name"
- `Input` for name (`required`, `rounded-none`)

**Section 2 — Services** (full-width):

**Section 3 — Pricing** (single column):
- Label: "Package Price"
- `Input type="number"` with "₱" prefix text, `min="1"`, `rounded-none`
- Help text: `text-xs text-muted-foreground` — "Set the final bundle price customers will pay"
- Savings preview (auto-calculated from selected services total minus package price)

**Section 4 — Featured Image** (full-width):
- Reuse the Vercel Blob upload pattern from `ExhibitForm.tsx`
- Drop zone for image upload, preview thumbnail if image exists
- Store `image_url` from Vercel Blob response

**Section 5 — Settings** (2-column grid at `sm:`, 1-column mobile):
- `display_order` — `Input type="number"` with label "Display Order"
- `valid_from` — `Input type="date"` with label "Available From" and help "Leave empty for always available"
- `valid_until` — `Input type="date"` with label "Available Until"
- `max_redemptions` — `Input type="number"` with label "Redemption Limit" and help "Leave empty for unlimited"
- `is_active` — `Switch` with label "Active"

**DialogFooter:**
- Cancel: `Button variant="outline" className="rounded-none h-12 px-6 uppercase tracking-widest text-[10px] font-bold"`

  - Label: "Create Package" (create) / "Update Package" (edit)
  - Loading: `Loader2 className="animate-spin"` + "Creating..." / "Saving..."
  - Disabled when: `submitting || selectedServiceIds.length < 2 || !name || !price || price <= 0`

- On edit: call `updatePackage(id, payload)` from apiClient
- On success: toast "Package saved successfully.", call `onSuccess()`, close dialog
- On error: toast "Failed to save package. Please try again."

**Edit mode pre-population:**
- If `editPackage` is provided, populate all fields from it
- `selectedServiceIds` = `editPackage.services.map(s => s.id)`

<action>
Create `frontend/src/components/packages/PackagesView.tsx`:

**Component structure:**
1. **Header** — Flex row:
   - Heading: `font-serif text-2xl font-light` — "Package Studio"
   - Right-aligned: `Button` — "Create Package" with `Plus` icon

2. **Package list** — `useQuery` fetching `getPackages()` from apiClient:
   - Loading state: Skeleton shimmer placeholders
   - Error state: `"Failed to load packages."` inline alert
   - Empty state: Centered layout with `Package` icon (48px, muted), heading `font-serif text-xl font-light` — "No packages yet.", body `text-sm text-muted-foreground` — "Create your first service bundle to increase ticket size and simplify booking.", CTA Button — "Create Your First Package"

3. **Package cards** — Vertical stack, each Card with:
   - Layout per UI-SPEC card anatomy: `[64x64 img] | Package Name + meta | Price + actions`
   - Image: `64x64px rounded-none object-cover`, fallback: `bg-primary/5` with `Package` icon centered
   - Package name: `font-serif text-lg font-light`
   - Service count: `text-[10px] uppercase tracking-widest font-bold text-muted-foreground` — "{N} services"
   - Bookings count: `text-[10px] text-muted-foreground` — "{N} bookings"
   - Status: `Badge variant="default"` (green) for active, `Badge variant="outline"` for inactive
   - Validity: `text-xs text-muted-foreground` — "Valid: {start} – {end}" or "Always available"
   - Price: `font-serif text-lg font-light text-primary` — "₱{price}"
   - Edit: `Button variant="ghost" size="icon"` with `Pencil` icon
   - Toggle: `Button variant="ghost" size="icon"` with `ToggleLeft`/`ToggleRight` icon
   - Delete (only when `bookings_count === 0`): `Button variant="ghost" size="icon"` with `Trash2` icon
   - Card hover: `hover:bg-primary-ultra/10 transition-colors`

4. **CRUD handlers:**
   - Edit click → open `PackageBuilderDialog` with `editPackage` prop
   - Toggle click → `window.confirm("Deactivate '{name}'? It will no longer be available for booking.")` then call `togglePackage(id, !is_active)` + invalidate query
   - Delete click → `window.confirm("Permanently delete '{name}'? This cannot be undone. Existing bookings are not affected.")` then call `deletePackage(id)` + invalidate query
   - Create/edit success → invalidate packages query (React Query `queryClient.invalidateQueries`)
</action>
