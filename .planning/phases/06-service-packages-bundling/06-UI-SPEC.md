## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chip internal padding |
| sm | 8px | Compact element spacing, service chip gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, form group spacing |
| xl | 32px | Layout gaps between package cards |
| 2xl | 48px | Major section breaks, package hero padding |
| 3xl | 64px | Page-level spacing (services page package band) |

Exceptions:
- Package builder form fields: 12px vertical gap (gap-3) between label+input pairs — matches Phase 5 CMS pattern
- Service selection chips: 8px gap between chips, 4px internal padding
- Package card featured image: 0px padding (full-bleed within card)
- Touch targets: minimum 44px height for "Book Package" CTA on mobile

## Layout Contract

### Package Management — "Packages" View (D-11, Agent Discretion: sidebar placement)

**Location:** New sidebar nav item in ManagerDashboard, registered as `activeView: 'packages'`

**Structure:** Full-width content area (inherits Phase 4 sidebar+content layout)

| Zone | Behavior |
|------|----------|
| Header | Flex row: serif heading "Package Studio" + "Create Package" primary CTA (right-aligned) |
| Package list | Vertical stack of package Cards, each showing: featured image thumbnail (64x64), name, price, service count, status badge, edit/deactivate actions |
| Empty state | Centered icon + heading + CTA when no packages exist |

**Sidebar entry:**
- Icon: `Package` from lucide-react
- Label: `Packages`
- Position: After "Content" in nav order (last operational item before system footer)
- Active state: `bg-primary text-white shadow-lg shadow-primary/20` (same as all other items)

## Interaction Patterns

### Package Builder Flow (Manager)
1. Manager clicks "Create Package" → PackageBuilderDialog opens
2. Fill basic info: name + description
3. Select services via ServiceChipSelector: click chips to toggle, category-grouped, summary updates live
4. Set package price in ₱ input → savings preview auto-calculates
5. Optionally upload featured image (same Vercel Blob pattern as ExhibitForm)
6. Configure settings: display_order, validity dates, max_redemptions, is_active toggle
7. Click "Create Package" → `Loader2 animate-spin` + disabled state
8. On success: toast "Package saved successfully." + dialog closes + list refreshes (React Query invalidation)
9. On error: toast "Failed to save package. Please try again."

### Package Edit Flow (Manager)
1. Manager clicks `Pencil` icon on package card → PackageBuilderDialog opens pre-populated
2. Modify any fields → click "Save Changes"
3. Same loading/success/error behavior as create

### Package Deactivation (Agent Discretion: soft deactivate)
**Decision:** Soft deactivate only — no hard delete for packages with any booking history. Packages with 0 bookings may be permanently deleted.
1. Manager clicks toggle icon → `window.confirm("Deactivate '{name}'? It will no longer be available for booking.")`
2. On confirm: API PATCH `is_active: false` → card status badge changes to "Inactive" (outline variant)
3. Inactive packages remain in list but are hidden from customer-facing discovery
4. Delete option shown only for packages with `bookings_count === 0`

### Customer Package Discovery
1. Customer navigates to `/services` → packages section loads at top (if any active packages exist)
2. Cards show featured image, name, included services, pricing with savings
3. Click "Book Package" → package added to cart as single item with child services
4. CTA changes to "View in Cart" (green success variant) → click navigates to `/booking`

### Cart Package Booking (D-04)
1. Package enters cart as single `CartItem` with `type: 'package'` and `childServices` array
2. On Booking page: parent header shows package name + price, child services expand below
3. Customer selects staff + time for EACH child service independently (same UX as individual services)
4. Remove button removes entire package (all child services) from cart
5. Summary sidebar shows package name with child services indented, total uses package price

### Package + Individual Service Mixing (Agent Discretion)
- Cart can contain both packages and individual services simultaneously
- No restrictions on combinations
- Each item (package or individual) requires complete staff/time selection
- Total price = sum of all package prices + sum of all individual service prices

### Inline Performance Stats (Agent Discretion)
**Decision:** Show minimal inline stats — booking count only. Revenue analytics deferred to Phase 7.
- Each package card in manager view shows: `text-[10px] text-muted-foreground` — "{N} bookings" after status badge
- No revenue display in this phase
