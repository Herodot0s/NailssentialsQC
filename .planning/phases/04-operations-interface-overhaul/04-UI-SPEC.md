## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell gutters |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, sidebar item vertical rhythm |
| xl | 32px | Layout gaps between major card groups |
| 2xl | 48px | Major section breaks, header-to-content gap |
| 3xl | 64px | Page-level spacing (section bands) |

Exceptions:
- Sidebar nav items: 12px vertical padding (py-3) for compact density
- Touch targets: minimum 44px height for mobile check-in button
- Overview stat cards: 32px internal padding (p-8) for spacious feel per D-08

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display | 36px | 400 (regular) | 1.2 |
| Heading | 24px | 400 (regular) | 1.3 |
| Body | 14px | 400 (regular) | 1.5 |
| Label | 10px | 700 (bold) | 1.2 |

**Font families:** Inter (sans-serif body/labels), Playfair Display (serif display/headings)

**Weights:** 400 (regular — body, display via `font-light` serif) and 700 (bold — labels, micro-copy)

**Established conventions (from existing codebase — CSS utility overrides, not additional tokens):**
- Labels use `text-[10px] tracking-widest uppercase font-bold` — signature NailssentialsQC pattern
- Headings use `font-serif` (Playfair Display) with `font-light` (maps to weight 400 display)
- Micro labels (`text-[9px]`, `text-[8px]`) are CSS overrides of the Label token for compact contexts
- Shift times use `font-mono text-xs` (inherits Body size with monospace override)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `hsl(var(--background))` | Page background, main content area, card surfaces |
| Secondary (30%) | `#fafaf9` / `var(--bg-color)` | Sidebar background on hover, table header rows, muted card fills |
| Accent (10%) | `#FF385C` / `var(--brand-rausch)` | Active sidebar item background, primary CTA buttons, progress bars, active tab underline |
| Destructive | `hsl(var(--destructive))` | Delete confirmations, absence deductions, negative financial amounts |

Accent reserved for:
- Active sidebar navigation item (bg-primary text-white)
- Primary action buttons ("New Employee", "Run Weekly Payroll")
- Progress bar fills (revenue target, commission engine)
- Active tab underline indicators
- Status card left-border accent stripe
- Mobile check-in swipe button (default variant)

**Additional semantic colors (established):**
- Success: `var(--success-color)` #4caf50 — checked-in status, positive trends, "Finalize Ritual" button
- Info: `var(--info-color)` #2563eb — active/in-progress appointment status
- Warning: `var(--warning-color)` #ff9800 — tardiness alerts
- Muted foreground: `hsl(var(--muted-foreground))` — inactive sidebar items, secondary text

## Layout Contract

### Manager Dashboard (D-01, D-02, D-04, D-05, D-08, D-09)

**Structure:** Persistent sidebar (w-64, 256px) + scrollable content area

| Zone | Width | Behavior |
|------|-------|----------|
| Sidebar | 256px fixed | Sticky `top-0 h-screen`, white bg, border-r border-gray-100 |
| Content | flex-1 | Scrollable, p-10 (40px padding) |

**Sidebar anatomy (top to bottom):**
1. Brand block (p-8, border-b): "NailssentialsQC" in `font-serif text-xl` + "Enterprise Suite" micro-label
2. Nav items (p-4): Icon + label buttons, `text-[10px] uppercase tracking-widest font-bold`
3. System status footer (p-6, border-t): Settings icon + "Live" pulse indicator

**Sidebar active state:** `bg-primary text-white shadow-lg shadow-primary/20`
**Sidebar inactive state:** `text-muted-foreground hover:bg-gray-50 hover:text-foreground`

**Overview landing (D-04):** 4-column stat card grid at lg breakpoint, 2-col at md, 1-col at mobile

**Density approach (D-08):**
- Overview/summary pages: Spacious — large serif headings, big stat numbers (text-4xl font-serif), generous gap-8/gap-12
- Drill-down sections: Compact — operational tables with tight row padding, dense data display

### Staff Dashboard (D-03, D-06, D-07)

**Structure:** Container layout (max-w-7xl mx-auto), tab-based navigation

| Zone | Behavior |
|------|----------|
| Header | Flex row, "Artisan Terminal" branding + action buttons |
| Tabs | Schedule / Earnings / Messages — underline-style tabs |
| Content | Tab-specific content, space-y-12 between sections |

**Tab style:** `text-[10px] uppercase tracking-[0.2em] font-bold`, border-b-2 active indicator in primary color

**Check-in card (D-06):**
- Desktop: lg:col-span-4 in 12-column grid, gradient background `from-primary/5 to-secondary/5`
- Mobile: Full-width, prominent SwipeButton component
- Clock display: 56px (text-6xl) font-serif in centered circular container (w-56 h-56)
- Status badge: rounded-none, success-color when active

**Appointments display (D-07):**
- Desktop: lg:col-span-8 table view with Timeline/Client/Treatment/Status/Actions columns
- Table header: `bg-muted/30`, `text-[9px] uppercase tracking-widest font-bold`
- Row hover: `hover:bg-primary-ultra/10 border-primary/5 transition-colors`

### Responsive Breakpoints

| Breakpoint | Manager Sidebar | Staff Layout |
|------------|----------------|--------------|
| < 768px (md) | Hidden, hamburger menu | Stack to single column, check-in dominates |
| 768-1024px (lg) | Collapsed icon-only (48px) | 2-column grid |
| > 1024px | Full sidebar (256px) | 12-column grid |

## Interaction Patterns

### Sidebar Navigation (Manager)
- Click nav item → instant content swap via React state (`activeView`)
- No page reload, no route change (internal state navigation)
- Active item: solid primary background with white text + shadow
- Hover: subtle gray background lift

### Staff Check-in Flow (D-06)
1. Dashboard loads → check attendance status
2. If not checked in: SwipeButton dominates ("Swipe to Initialize Shift")
3. After check-in: SwipeButton changes to destructive variant ("Swipe to Check Out Artisan")
4. Status badge updates: "Off Duty" → "Status: Active" with green pulse

### Staff Detail Sheet (Manager)
- Click staff row → Sheet slides from right (sm:max-w-xl)
- Header: primary background with avatar, name, badge
- Body: Tabbed content (Profile / Schedule / Financials)
- Tab underline: primary color border-b-2

### Data Tables
- Row hover: `bg-primary-ultra/10` subtle highlight
- Sortable columns: not required this phase (deferred)
- Pagination: not required (existing API patterns handle this)
