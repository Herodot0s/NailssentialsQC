## Task 1: Create SliderCheckIn Component

<read_first>
- frontend/src/pages/StaffDashboard.tsx (see existing checkIn/checkOut API imports and usage)
- frontend/src/components/ui/card.tsx (for consistency in component patterns)

<objective>
Create a reusable SliderCheckIn component that replaces the button-based check-in/out in StaffDashboard.
</objective>

<action>
Create `frontend/src/components/SliderCheckIn.tsx` with:

1. Props interface:
   - `isCheckedIn: boolean` — current check-in status
   - `checkInTime: string | null` — last check-in time to display
   - `onCheckIn: () => void` — callback for check-in
   - `onCheckOut: () => void` — callback for check-out
   - `currentTime: Date` — current time to display

2. Component state:
   - `sliderX` (number, 4-56 range) — thumb horizontal position
   - `isDragging` (boolean) — track drag state
   - `startX` (number) — drag start position

3. Visual design (matching Variant B sketch and project brand):
   - Container: card-like div with `rounded-none` (project convention), border `border-primary/10`
   - Status indicator: colored dot (8px circle) + text "Off Duty" (gray) or "Active" (success green #4a8c6f)
   - Clock: font-serif, 42px, font-weight 300, displaying `currentTime` in `h:mm` format
   - Label: uppercase, 11px, tracking-wide, muted color, "Current Time"
   - Track: 100% width, 64px height, `rounded-full`, background `var(--color-primary-light)` (cream), border `border-primary/10`
   - Fill: absolutely positioned left-0 bottom-0, height 100%, width matches thumb position + 56px, background `bg-primary` (#B8794E), `rounded-full`, transition `width 0.15s ease`
   - Thumb: 56x56px, white, `rounded-full`, `shadow-md`, border `border-2 border-primary`, centered vertically, left position via `sliderX`, cursor grab, transition `left 0.15s ease`, touch-action none
   - Arrow icon in thumb: chevron-right SVG, 22px, color `text-primary`
   - Track label: centered text "Slide to Check In" (11px, font-bold, uppercase, tracking-wide, color primary), hidden when dragging
   - Info row: left side shows "In at {time}" or "Out at {time}", right side shows "Today: Xh Ym"

4. Interaction logic:
   - mousedown/touchstart on thumb: set dragging true, record startX, disable transition on thumb
   - mousemove/touchmove: calculate newX = clamp(clientX - startX, 4, 56). Update sliderX.
     - Update fill width to sliderX + 52
     - Change track label: if sliderX > 44 → "Release to Confirm", else "Slide to Check In" / "Slide to Check Out" based on isCheckedIn
     - Change track bg: if sliderX > 30 → success-light, else primary-light
   - mouseup/touchend: set dragging false, re-enable transitions
     - If sliderX > 48 (85% threshold): trigger check-in or check-out
       - If !isCheckedIn → call onCheckIn(), set sliderX = 56 (full right)
       - If isCheckedIn → call onCheckOut(), set sliderX = 4 (full left)
     - Else: animate back to starting position (56 if checked in, 4 if not)
   - Update fill and track bg accordingly

5. Animation: CSS transitions on thumb (left 0.15s), fill (width 0.15s). No spotlight animation (cleaner for this project).

6. Use inline styles for track/fill (since they're dynamic), Tailwind for everything else.

7. Export as default.
</action>
