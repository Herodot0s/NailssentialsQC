## Root Cause Analysis

The `DropdownMenuTrigger` in `dropdown-menu.tsx` (line 18-21) uses `@base-ui/react/menu` but imports `Slot` from `@radix-ui/react-slot`. When `asChild` is true, it renders:

```tsx
const Comp = asChild ? Slot : MenuPrimitive.Trigger;
return <Comp data-slot="dropdown-menu-trigger" {...props} />;
```

Radix's `Slot` merges its props (onClick, ref, data-state, etc.) into the child component. However, the child is the `Button` from `button.tsx`, which uses Base UI's `ButtonPrimitive` with a `render` prop pattern:

```tsx
function Button({ render, ...props }) {
  return (
    <ButtonPrimitive
      render={render}
      {...props}
    />
  );
}
```

When Radix's `Slot` tries to merge trigger props onto this Button, the `render` prop is not properly handled, and the trigger event listeners are not attached to the actual DOM button element.

## Fix Strategy

Replace the `Button` component inside `DropdownMenuTrigger asChild` with a standard HTML `<button>` element that:
1. Properly forwards all props from Radix's `Slot`
2. Maintains the same visual styling as the current avatar button
3. Works for both the desktop avatar dropdown and the mobile menu dropdown
